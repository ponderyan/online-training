# 🦊 FoxLearn · 狐学 — 产品架构说明书

> 版本：v1.0 · 2026年6月17日
> 定位：完整在线培训考试平台 · 产品模块划分与数据关系定义

---

## 一、总览：五大模块

```
┌────────────────────────────────────────────────────────────┐
│                        FoxLearn                             │
├──────────┬──────────┬──────────┬──────────┬───────────────┤
│ 资源中心  │ 教务中心  │ 学习中心  │ AI引擎    │ 系统管理      │
│ Resource  │ Academic │ Student  │ AI       │ System        │
├──────────┴──────────┴──────────┴──────────┴───────────────┤
│  内容创建      组织执行      学员端       能力注入      基础设施       │
│  出题人视角    教务视角      学员视角     无独立UI      管理员视角      │
└────────────────────────────────────────────────────────────┘
```

**设计原则：**
- 每个模块是内聚的功能集合，不是按开发顺序切分
- AI引擎不是独立页面，而是渗透到其他模块的水平能力
- 数据模型与模块一一对应，跨模块关联通过外键+引用ID

## 四、数据模型变更清单

### 需新增的模型

| 模型 | 所属模块 | 说明 |
|------|---------|------|
| Exam | 教务中心·考试管理 | 考试场次：名称、时间、模式、关联试卷、防作弊配置 |
| ExamAssignment | 教务中心·考试管理 | 学员分配：学员ID、考试ID、状态、开始/交卷时间 |
| ExamAnswer | 学习中心+教务中心 | 逐题答题记录：题snapshot、答案、得分、用时 |
| ExamResult | 教务中心·判分 | 成绩汇总：总分、客观分、主观分、状态、调整分 |
| ScoreAdjustmentLog | 教务中心·判分 | 成绩调整审计日志：原始→调整→谁→为什么→时间 |
| ProctorLog | 教务中心·监考 | 监考日志：异常记录、干预操作 |
| CertificateRule | 教务中心·认证 | 认证规则：模式、有效期、补考配置 |
| Certificate | 教务中心·认证 | 学员证书：颁发、到期、状态 |
| Course | 资源中心·课程 | 课程：名称、简介、封面、类型、学时 |
| CourseLesson | 资源中心·课程 | 课时：标题、视频、时长、顺序 |
| CourseInstructor | 资源中心·课程 | 讲师关联 |
| TrainingClass | 教务中心·班级 | 培训班：升级自StudentGroup |
| TrainingClassCourse | 教务中心·班级 | 培训班-课程关联 |
| CourseProgress | 学习中心 | 学员学习进度 |
| PracticeRecord | 学习中心·练习 | 练习记录 |

### 需扩展字段的现有模型

| 模型 | 新增字段 | 说明 |
|------|---------|------|
| Question | `group` (PRACTICE/EXAM/COMMON) | 题目分组归属 |
| Paper | `examCount` (引用统计) | 试卷被多少个考试场次引用，定稿后不可修改 |
| User | `tenantId` (预留) | 多租户，本期可为 null |
| Material | `courseId` (可选) | 教材关联到课程 |

## 七、设计哲学

1. **先会话再持久** — 所有设计先讨论确认，不动代码
2. **现有代码不动筋骨** — 只加字段不拆表，只改导航不改逻辑
3. **模块内聚** — 功能按业务域组织，不按时序
4. **AI是水平能力** — 不单独开页面，出现在它该出现的地方
5. **架构预留** — 多租户、高级考试模式、AI引擎在设计层面存在，实现层面渐进
6. **记录驱动** — 一切操作可追溯（成绩调整、监考干预、权限变更）
